# Security Policy

VDS Guardian is currently pre-release and must not be relied on as the only
backup or disaster-recovery mechanism.

Please do not open a public issue for a suspected vulnerability. Contact the
maintainers privately through a GitHub security advisory at
https://github.com/ProAnima/VDS-Guardian/security/advisories/new, and include a
minimal reproduction, affected commit, impact, and suggested mitigation.

Never send private keys, production manifests, server addresses, or unredacted
diagnostic bundles in a report. The project threat model is documented in
`docs/SECURITY_MODEL.md`.
